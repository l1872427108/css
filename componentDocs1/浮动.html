# 浮动

# 概念

当元素浮动以后可以向左或者向右移动，直到它的外边缘碰到包含它的框或者另一个浮动元素的边框为止。元素浮动会脱离正常的文档流，所以文档的普通流中的框就变得好像浮动元素不存在一样。

# 优点

可以在图文混排的时候可以很好的使文字环绕在图片周围。当浮动之后，有着块级元素的一些特性可以设置宽高等。

# 于 inline-block 的区别

横向排列的时候，float 可以设置方向，inline-block 是固定的。

inline-block 有时会有空白间隙。

# 缺点

浮动元素一旦脱离文档流，无法撑起父元素。造成父亲元素的高度塌陷。

# 清除浮动的方式

  1. 额外标签法
.clear {clear: both;}
<div>
    <div></div>
    <div></div>
    <div class="clear"></div>
</div>

添加了没有意义的标签，结构化较差。

添加的必须是快元素

  1. 父亲添加 overflow 实行，或者设置高度

<div style="overflow:hidden/auto">
    <div></div>
    <div></div>
    <div class="clear"></div>
</div>

无法显示溢出的部分.

  1. 伪类选择器清楚浮动
.parent::after {
    // 设置添加子元素的内容是空的。
    content: '';
    // 设置添加子元素为块元素
    display: block;
    // 设置添加的子元素高度 0
    height: 0;
    // 设置添加子元素看不见
    visibility: hidden;
    // 设置 clear
    clear: both;
}
.parent {
    /* 兼容 IE6、7 专有 */
    *zoom: 1;
}
